数据库基础 - SQL tips

发表于 2017-05-07 | 更新于 2017-05-21 | 分类于 Programming ， SQL |

一些小的SQL语言tips

结束SQL语句：多条SQL语句必须以分号(;)分隔

大小写规范：通常，所有关键字大写，所有列和表名小写

使用通配符：通配符会降低检索速度，所以尽量列明需要检索的列

升降续：ASC，DESC

子句的位置顺序：

子句中圆括号的使用：在任何使用AND和OR操作（及其他必要的）子句中，都应该使用圆括号明确分组操作，不要过分依赖默认计算机次序。而且这有助于消除歧义。

通配符%：%可以通配绝大多数内容，除了NULL。

总是使用DataDiff()：在比较两个日期时，应该总是使用DataDiff()，而且不要假定日期是如何存储的。

AVG只用于单个列：为了获得多个列的平均值，必须使用多个列

在多个列上进行计算：利用标准的算是操作符，所有聚集函数AVG/COUNT/SUM/MAX/MIN都可以用来执行多个列上的计算。

NULL值：AVG/MAX/MIN/SUM忽略值为NULL的行；当使用count()时，如果指定列明NULL值将被忽略，如果count(*)则NULL值不被忽略

Distinct：distinct行级的，必须放在所有字段开头。distinct如果跟group by 一起使用，查询的效率会高一些，比如select *, count(distinct name) from table group by name。虽然group by也有这种效果，但是，group by的主要用途还是分组查询。

WHERE与HAVING：两者都是对于数据进行过滤，不同点在于WHERE是进行过滤行，HAVING是进行过滤分组；WHERE是在分组前进行过滤，HAVING是在数据分组后进行过滤。

配套：HAVING都总是与GROUP BY 配套的，GROUP BY总是与ORDER BY配套的。

当使用GROUP BY时，SELECT只能选GROUP BY后唯一的字段，不可以跟不唯一的字段。

TOP最好当作函数使用，写作TOP(1)而不是TOP 1

子查询与多表查询：多数的子查询可以通过多表查询来得到，但是子查询的逻辑会更清楚。

联结的表越多，性能下降越厉害。处理联结表示非常耗费资源的，因此不要联结不必要的表。通过为外键列有效地创建索引可以明显改善这种性能下降。

IN/EXISTS, WHERE, INNER JOIN通常可以替换使用。
